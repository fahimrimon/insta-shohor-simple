Q-1: How Does Javascript Work?

At first we need to know what is Javascript?
We know, Javascript is a scripting and text based programming language. It is used to client side and server site that is allows and make our webpage interactive.
In client side, Javascript support modern web browser and have their Javascript engine. We know google chrome ahs its own Javascript engine called V8. Javascript engine totally depends on the web developers how developers use Javascript, because it can be used for several things in web development. At first we created a simple HTML document then we added javascript code in it. Our js code always handles and runs Javascript engine, If we have used a chrome browser to run our any program that has the v8 engine that is also create the node.js. We know Javascript is a interpreted language, so it is executed in line by line manner so it's means Javascript engine always converts the js code line by line and runs in the same manner.
Javascript engine work: Javascript file -> Parser -> AST -> Conversion to Machine code -> Machine code -> Finally Code runs.

Q-2: Defference between Local Storage and Session Storage.

Local Storage:
  1. Data deleted, when user delete menually.
  2. Storage Capacity maximum 5MB-10MB.It's browser depended.
  3. Accessibility all window.
  4. Scope- Single domain, No sub domain.
Session Storage:
  1. Data deeleted, when browser is closed.
  2. Storage Capacity maximum 5MB.
  3. Accessibility Private to tab.
  4. Scope- Single domain, But it's per bowser tab.
